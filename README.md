# HouseMaster — Pavlodar Thermal Map (MVP v0.1)

Interactive GIS MVP for the HouseMaster / Pavlodar thermal-supply evidence
project. This is a **running application**, not a specification or a
presentation — the source-of-truth documents in the parent project
(`TASK 01–09`) were used as input and were not modified.

## Run it

```bash
npm install
npm run dev       # local dev server (Vite)
npm run build     # production build → dist/
npm run preview   # serve the production build locally
npm test          # vitest — unit + smoke tests
npx tsc --noEmit -p tsconfig.app.json   # typecheck
```

Requires Node 18+ (developed/tested on Node 22).

## What's real vs. demo

The app deliberately keeps two data contours apart, per the task brief:

- **Verified / public-context data**: the three heat sources (ТЭЦ-1/2/3) and
  the TM-20A project line. Coordinates for the heat sources were **not**
  available from an official akimat/PTS cadastral source in the attached
  evidence pack, so they were looked up in a public map reference
  (Global Energy Monitor's Global Coal Plant Tracker, gem.wiki) and are
  tagged accordingly:
  `location_status: public_map_reference`, `verification: not_officially_confirmed`.
  See `src/data/heatSources.ts` for the exact source URLs and dates.
  TM-20A is rendered as a **schematic** line only
  (`geometry_status: schematic`) between those points — it is not a
  cadastral or as-built trace. See `src/data/tm20a.ts`.

- **Synthetic demonstration buildings**: 24 buildings, IDs `DEMO-PVL-001`
  through `DEMO-PVL-024`, generated by `scripts/generate-demo-buildings.mjs`
  into `src/data/demoBuildings.json`. They are not tied to real addresses,
  carry no personal data, and are flagged `demo: true` everywhere the UI
  reads them — the map, the legend, and every Building Card show
  **"DEMO DATA — NOT A REAL BUILDING ASSESSMENT"** persistently.

No real building in this MVP is presented as officially inspected, and no
numeric thermal deficit or real address is invented, per the task's hard
data constraint.

## The three-status model

Every building carries three **independent** axes — Knowledge, Risk, Action —
per `DESIGN_SYSTEM_PRINCIPLES.md`. They are never merged into a single
"good/bad house" color. The map fill color encodes Knowledge, the marker
ring encodes Risk, and Action is shown as a separate badge in the Building
Card and Legend. Red is reserved for a confirmed `specialist_required` risk
status only — there is no "red map of bad houses."

## Import / export

`Импорт` accepts `.geojson`, `.json` (array or `{buildings:[...]}`), and
`.csv`. Rows are validated with Zod (`src/schemas/buildingSchema.ts`);
missing Building ID, invalid coordinates, unknown enum values, and duplicate
IDs are all surfaced as row-level errors before anything is added to the
session. Nothing is persisted to a server. Sample files (including one with
intentional errors) are in `public/sample-data/`.

`Экспорт` writes only the "safe" public-summary fields (no raw locators, no
free-text notes) as GeoJSON or CSV, matching `DATA_PROVENANCE_UI.md`'s
public-summary rule.

## Verification performed in this environment

- `npx tsc --noEmit` — passes, no type errors.
- `npx oxlint src` — 0 warnings, 0 errors.
- `npm test` (vitest) — 22 tests passing: import validation (valid/invalid
  GeoJSON & CSV, duplicate IDs, unknown enums, bad coordinates, unsupported
  formats), filter semantics (AND across groups / OR within group, demo
  toggle, search), dashboard metrics (computed, not hardcoded), demo dataset
  integrity (24 unique `DEMO-PVL-###` IDs inside a plausible Pavlodar
  bounding box), and an App-level smoke test (header, view switch, filter
  panel, import modal all render and respond to interaction).
- `npm run build` — production build succeeds (`dist/`); `npm run preview`
  was started and the served HTML/JS/CSS/sample-data files were confirmed
  reachable over HTTP (200 responses via curl).
- **Not possible in this sandbox**: pixel-level visual screenshot QA in a
  real browser. There is no GPU/WebGL-capable browser available here
  (no Chromium binary, and Playwright/Puppeteer browser downloads are
  outside the network allowlist), so MapLibre's actual WebGL rendering was
  not visually inspected — only exercised via a mocked-map smoke test and
  manual code review. Please do a quick visual pass with `npm run dev` /
  `npm run preview` on your own machine before treating the map rendering
  as fully verified.

## Known follow-ups (not blockers for this MVP)

- The production JS bundle is ~1.37 MB (~369 KB gzipped), mostly MapLibre
  GL; code-splitting could reduce initial load if needed later.
- Layers `network_mains`, `connection_points`, `restriction_zones`,
  `modernization_projects`, `mv`, `renovation` are wired into the Layer
  Panel and toggle correctly but intentionally carry no geometry — per the
  brief, they show "Данные ожидаются после институционального
  согласования" rather than an error or fabricated shapes.
